# BUILD TASK: netviz — Network Diagram Automation & Documentation Tool

You are building a complete, working application in this session. Read this entire document before writing any code.

## Mission

Build an interactive network topology documentation tool as a **single self-contained HTML file** (`netviz.html`). It ingests link/node data (CSV/JSON/YAML), renders an interactive graph with rich visual encoding, supports full in-app editing, and exports images, data files, and documentation reports. No backend. No build step. It must open directly via `file://` and work offline after first load.

## Locked Technical Decisions — do not deviate or ask

1. **Single HTML file**: all CSS and JS inline. The ONLY external dependencies are CDN `<script>` tags for the libraries below. Everything else is vanilla JS (ES2020+). No React, no bundler, no npm.
2. **Graph engine**: Cytoscape.js (latest 3.x from cdnjs) + `cytoscape-fcose` layout plugin.
3. **YAML parsing**: js-yaml from cdnjs.
4. **CSV parsing**: PapaParse from cdnjs.
5. **Image export**: Cytoscape's built-in `cy.png()`/`cy.jpg()`; SVG export via the `cytoscape-svg` plugin if available on cdnjs, otherwise omit SVG export and note it in the README.
6. **No localStorage/sessionStorage for anything critical.** Use an in-memory state object. Offer explicit "Save Project" (downloads JSON) and "Open Project" (file input). You MAY use localStorage as an optional convenience autosave wrapped in try/catch with full functionality when unavailable.
7. **State management**: one global `AppState` object (project data + UI state) with a simple pub/sub or explicit `render()` calls. An undo/redo stack of deep-cloned project snapshots, capped at 50.
8. **Dark theme default**, light theme toggle. Professional look: think network ops tool, not consumer app. System font stack, no icon fonts — inline SVG only.

## Build Order (implement in phases; verify each phase before the next)

**Phase 1 — Core rendering.** Data model, hardcoded sample project in JS, Cytoscape rendering with full visual encoding (§3), legend, zoom/pan/fit, hover tooltips, click detail panel (read-only), dark theme.
**Phase 2 — Import.** CSV import with header-mapping wizard and validation report; JSON/YAML project import; merge modes (replace / merge-add / update-matching).
**Phase 3 — Editing.** Detail panel becomes editable; add/delete nodes and links; draw-link mode; right-click context menu; box-select bulk edit; undo/redo; manual drag with position persistence.
**Phase 4 — Layout & filters.** fcose, hierarchical (breadthfirst by `layer`), circular-per-site, manual; compound site grouping with collapse/expand; search with highlight-and-zoom; composable filters (site, layer, link_type, vlan, media, status); overlay modes (physical-only / tunnels-only / combined); VLAN coloring overlay.
**Phase 5 — Export & docs.** PNG (1x/2x/4x, transparent or themed bg, optional title block + legend baked in), JPEG, JSON, YAML, CSV round-trip, and the HTML documentation report (embedded diagram image + link inventory table + per-site node inventory).
**Phase 6 — Polish & verify.** Icon mapping settings, per-node icon upload (base64 into project file), keyboard shortcuts (Ctrl+Z/Y, Ctrl+S, Del, F, /), light theme, issues/validation panel, run ALL acceptance checks, write TESTING.md.

## Data Model

### Node

`id` (unique, auto from name if omitted), `name`, `type` (enum: router, l3-switch, l2-switch, firewall, server, hypervisor, access-point, wlc, load-balancer, storage, camera, printer, workstation, cloud, internet, isp, unknown), `mgmt_ip`, `site`, `layer` (core|distribution|access|edge|dmz|wan), `vendor`, `model`, `icon` (icon key or `custom:<id>`), `notes`, `x`, `y` (null = auto-layout).

Nodes referenced by links but not defined explicitly are auto-created with `type: unknown`.

### Link

`id` (auto if omitted), `source`, `target` (node names/ids), `source_if`, `target_if`, `speed` (10M|100M|1G|10G|25G|40G|100G), `media` (copper|fiber-sm|fiber-mm|dac|wireless|serial|virtual), `link_type` (physical|lag|ipsec|gre|mpls|sdwan|vpn-client|vxlan|logical), `stp_state` (forwarding|blocked|n/a), `vlan` (single, comma list, or range string), `subnet`, `status` (up|down|planned|decommissioned), `bandwidth_committed`, `provider`, `circuit_id`, `description`.

### Project JSON (canonical save format)

```json
{
  "meta": {
    "name": "",
    "schema_version": 1,
    "author": "",
    "created": "",
    "modified": ""
  },
  "settings": {
    "theme": "dark",
    "layout": "fcose",
    "icon_map": {},
    "custom_icons": {}
  },
  "nodes": [],
  "links": [],
  "groups": []
}
```

`custom_icons` holds base64 data URIs so projects are portable.

## Visual Encoding (implement exactly; legend must reflect this and be toggleable)

- **media → line style/color**: copper = solid #9aa0a6; fiber-sm = solid #fbbc04; fiber-mm = solid #ff8a00; dac = solid #00bfa5; wireless = dashed #4d9fff; serial = dash-dot #b39ddb; virtual = dotted (color by link_type below).
- **tunnel link_type colors** (dotted): ipsec #ab47bc, gre #8d6e63, mpls #66bb6a, sdwan #26c6da, vpn-client #ec407a, vxlan #7e57c2, logical #78909c.
- **speed → width**: 10M/100M = 1.5px, 1G = 2.5px, 10G = 4px, 25G = 5px, 40G = 6px, 100G = 8px.
- **stp blocked**: red ⛔-style badge rendered at the link midpoint (use a mid-edge label or overlay), line desaturated.
- **status**: down = #ef5350 with ✕ mid-label; planned = 40% opacity; decommissioned = hidden unless toggle enabled.
- **lag**: parallel double-line effect (or a thick line with a "2×"-style member badge if parallel rendering is impractical in Cytoscape) — pick one, document it.
- **Node icons**: build an inline SVG icon set for every node type (simple, clean, monochrome-with-accent, Cisco-topology-icon style, original artwork). Register them as data URIs in Cytoscape `background-image`. Selected node = accent ring.

## Interaction Requirements

- Hover link → tooltip (endpoints + interfaces, speed, media, vlan). Hover node → name, type, mgmt_ip, site.
- Click node/link → slide-in right panel, all fields editable, Save/Cancel, Delete button.
- Toolbar: New, Open, Save, Import CSV, Export ▾, Add Node, Draw Link, Layout ▾, Fit, Search box, Filters button, Legend toggle, Theme toggle, Undo/Redo, Issues panel badge.
- Search matches name, mgmt_ip, vlan, subnet, circuit_id → highlights and zooms to matches.
- Filters compose (AND across categories, OR within a category).
- Site groups render as Cytoscape compound parents; collapse via `cytoscape-expand-collapse` plugin if available on cdnjs, else implement simple hide-children + proxy-node collapse manually.

## Validation (warnings panel, never hard blocks)

On import and on edit, flag: duplicate link IDs; duplicate links between the same interface pair; unknown enum values (coerce to `unknown`/`n/a` and warn); malformed subnet/VLAN strings; self-loops; stp blocked on a subnet-bearing routed link; VLAN mismatch between the two ends of a trunk when both ends are defined; tunnel links whose two endpoints share no physical path (graph-connectivity check on physical links only).
